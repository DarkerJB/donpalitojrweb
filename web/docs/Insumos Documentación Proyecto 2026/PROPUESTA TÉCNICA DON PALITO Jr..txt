Presentación del Proyecto

La presente propuesta tiene como propósito diseñar, desarrollar y validar un sistema de comercio electrónico personalizado para la Cafetería Don Palito Junior, ubicada en el municipio de Sabaneta, Antioquia.

El proyecto busca transformar la forma en que el negocio se relaciona con sus clientes y administra sus operaciones internas, mediante una plataforma digital moderna, segura y adaptable a distintos dispositivos.
La solución permitirá a los clientes acceder fácilmente a productos, precios, promociones, reseñas y métodos de pago digitales, y a los administradores gestionar inventarios, ventas, usuarios y estadísticas desde una interfaz sencilla e intuitiva.

En un entorno donde la digitalización se ha vuelto esencial para la sostenibilidad de los pequeños negocios, esta propuesta representa una respuesta estratégica a la necesidad de Don Palito Junior de evolucionar sin perder su esencia tradicional.


Justificación del Proyecto

La Cafetería Don Palito Junior ha construido su reputación sobre la calidad de sus productos típicos colombianos (buñuelos, palitos de queso, cafetería y bebidas), pero el crecimiento actual del mercado exige un salto hacia la transformación digital.

Los hábitos de consumo han cambiado: hoy los clientes esperan acceder a los productos desde su celular, pagar digitalmente y recibir una experiencia personalizada.

El sistema propuesto permitirá a Don Palito Junior:

-	Expandir su alcance más allá del público local, conectando con nuevos clientes del sur del Valle de Aburrá.
-	Mejorar la experiencia de compra, gracias a un entorno web/móvil rápido, visualmente atractivo y seguro.
-	Optimizar la gestión operativa, centralizando inventarios, precios, ventas y reseñas.
-	Reducir costos y errores operativos, automatizando procesos actualmente manuales.
-	Fomentar la fidelización, mediante promociones exclusivas, reseñas, calificaciones y notificaciones de descuentos.

En suma, este proyecto no solo moderniza las operaciones de la cafetería, sino que la posiciona como un referente local de innovación gastronómica, sin perder su identidad artesanal ni su cercanía con los clientes.


Alcance de la Propuesta

La propuesta abarca el diseño, desarrollo, implementación y validación de un sistema integral dividido en los siguientes servicios:

1. Sistema para Clientes
-	Visualización de productos, precios y disponibilidad.
-	Carrito de compras con opciones de agregar/eliminar.
-	Pagos seguros mediante código QR.
-	Promociones y descuentos activos.
-	Sistema de reseñas y calificación.

2. Sistema para Administradores
-	Control de inventario, ventas y pedidos.
-	Gestión de usuarios y cuentas administrativas.
-	Actualización de precios, promociones y stock.
-	Generación de reportes y estadísticas visuales.
-	Validación de roles y permisos mediante token.

3. Servicios de Seguridad
-	Autenticación con token y roles diferenciados.
-	Recuperación de cuenta mediante preguntas de seguridad.
-	Registro de actividad y bloqueo automático por intentos fallidos.

4. Sistema de Reportes
-	Visualización gráfica de ventas, productos más vendidos y reseñas.
-	Descarga de reportes en formatos CSV o PDF.

5. Módulo Cloud y API
-	Integración con servicios en la nube para almacenamiento y despliegue.
-	API REST documentada con Swagger para pruebas y mantenimiento.

El sistema será multiplataforma (web y móvil), de uso intuitivo y adaptable a distintas resoluciones de pantalla.


Metodologías

Enfoque de Desarrollo – Metodología Ágil (Scrum): El proyecto se desarrollará utilizando Scrum, marco ágil que permite organizar el trabajo en iteraciones cortas denominadas sprints, asegurando entregas funcionales frecuentes y retroalimentación continua con el cliente. Cada sprint incluirá planificación, desarrollo, pruebas, revisión y retroalimentación directa de los señores propietarios.

Metodología de Diseño
-	Herramienta: Figma
-	Objetivo: Diseñar prototipos de alta fidelidad que definan una experiencia moderna, ordenada y coherente con la identidad visual de Don Palito Junior.
-	El diseño seguirá los principios de usabilidad, coherencia visual y accesibilidad.

Metodología de Análisis
-	Herramientas: PlantUML y Markdown
-	Se elaborarán diagramas de casos de uso, clases, secuencia, componentes y arquitectura, garantizando la trazabilidad entre los requisitos funcionales y la estructura técnica.

Metodología de Desarrollo Frontend
-	Web: HTML, CSS y JavaScript
-	Mobile: React y Android
-	La interfaz estará centrada en la experiencia del usuario (UX), priorizando velocidad, claridad visual y compatibilidad con múltiples dispositivos.

Metodología de Desarrollo Backend
-	Lenguaje: Python
-	Base de datos: PostgreSQL
-	Se implementará una arquitectura basada en controladores y servicios, con rutas seguras, validación de datos y control de sesiones mediante tokens.
-	La API se documentará con Swagger para garantizar mantenibilidad y transparencia técnica.

Metodología de Testing
-	Herramientas: Postman y Swagger.
-	Se realizarán pruebas funcionales, de integración, de seguridad y validación de endpoints.
-	Cada sprint cerrará con una revisión de calidad y pruebas manuales de interfaz y flujo.

Metodología de Control de Versiones y Despliegue
-	Plataforma: GitHub (repositorio privado)
-	Se mantendrá control de versiones mediante ramas separadas (main, dev, feature).
-	El despliegue y la documentación técnica de la API se alojarán en un entorno cloud con integración REST.
